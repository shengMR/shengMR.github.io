State

初步使用

Compose UI的数据变化通过可观察状态来实现，常用的有State, LiveData, StateFlow, Flow, Observable

  • ViewModel+LiveData+Activity

  • ViewModel + Compose + Activity

    observeAsState 可观察 LiveData<T> 并返回 State 对象，每当 LiveData 发生变化时，该对象都会更新。State<T> 是 Jetpack Compose 可以直接使用的可观察类型

可组合项的状态

使用remember在可组合项中创建内部状态

  • 添加状态

其他状态

Jetpack Compose 并不要求您使用MutableState<T>存储状态，Jetpack Compose支持其它可观察类型，不过在读取其它可观察类型之前，需要转换成State<T>，才能在发生状态变化时自动重组界面，其它可观察类型如下：

  • LiveData
  • Flow
  • RxJava2

注意：可变对象（ArrayList或者mutableListOf()）可能会导致看到不正确的数据，不可变数据不能由Compose观察，因为发生变化时不能进行重组，需要使用可观察的存储器（State<List>）和不可变的listOf()

注意事项

1，在组合中被remember记住的值在配置更改（如旋转）的时候会被忘记并重新创建，可以改用保存的实例状态，在配置更改的时候自动保存和恢复状态

  • savedInstanceState

    可组合函数 savedInstanceState 会返回MutableState，它会在配置更改时自动保存和恢复自身
